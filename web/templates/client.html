OAuth 2.1 Flow Summary

  1. Authorization Request: Client redirects user to authorization server with PKCE challenge
  2. User Authentication: User logs in at authorization server
  3. Authorization Grant: Authorization server redirects back with authorization code
  4. Token Exchange: Client exchanges code for access token using PKCE verifier
  5. Resource Access: Client uses access token to access protected resources
